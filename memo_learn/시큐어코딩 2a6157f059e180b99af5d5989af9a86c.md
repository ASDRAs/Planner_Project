### key 1. AI는 도구가 아니라 문화다.

MS/Google : 신규 추가되는 코드 30%이상이 AI도구로 작성

아마존 : AI가 개발한 79%가 수정 없이 병합

코파일럿 자체 통계 : 자바 코드의 61%가 코파일럿의 제안에서 나온다.

개발 프로세스가 AI의 중심이 되어야 한다.

**AI시대에 개발자가 집중해야 할 것**

1. 알고리즘보다는 아키텍쳐
    1. 프로그래밍 언어는 추상화 계층이 많아지는 방향으로 발전했다 → 사람이 이해하기 쉽게 변해왔다
    2. 전체 시스템의 아키텍쳐를 설계하고 이해하는 능력이 중요해지고 있다.
    3. 생산성은 증가, but 해결해야 할 문제의 복잡도도 비례하여 증가.
2. 실무보다는 결정하고 책임지는 역할
    1. 실무는 AI가, 판단/결정은 인간이.
    2. 판단과 결정을 위해 큰 그림을 알아야 한다!

Copilot != Autopilot

copilot은 커스텀 여지가 매우 넓다. → 프롬프트와 세팅에 따라 성능차이 극심

어렵다, 어떻게 사용하냐가 중요

모델비교 - GPT계, Claude계, Gemini계

### key 3. 코드 작성보다 무결성 보장이 더 중요하다.

개발속도 폭발적으로 올려주지만, 스파게티 코드가 되기 쉽다.

→ 검증 가능한 속도를 확보하는 것이 중요하다. 테스트로 무결성을 보장하는 것이 중요!

TDD(Test-First)

- 기본 사이클 : Red - Green - Refactor
    - Red - 실패하는 테스트 코드를 먼저 작성
    - Green - 테스트를 통과하는 최소한의 코드를 작성
    - Refactor - 코드 품질을 개선, 최적화

Spec-First

- spec을 먼저 정의하는 것이 중요하다→spec이 곧 prompt
- “이 테스트 전부 통과하도록 구현해줘” 라고 지시하면 됨.

전통 TDD

메서드, 클래스 단위로 작동을 정의, 유닛 테스트 작성으로 시작 단계.

이점 : 코드 안전망

spec-first(tdd+예제스펙)

도메인언어로 스펙 시나리오 정의(우리의 언어, 자연어로 정의한다.)하여 시작

ex. A팀장이 회식메뉴 선택을 제안, B사원이 주꾸미를 제안한다.

대상

사용자 가치, 비즈니스 룰

이점

요구사항 명세, 커뮤니케이션 동기화

소프트웨어 개발 비용이 훨씬 저렴해졌다. → 코드를 갈아엎기 쉬워졌다. 한달에 두번 세번씩 갈아엎을수있다.(단방향 → 양방향)

정적분석, 테스트코드

.github/prompts/implenemt.prompt.md

- 항상 “git add .”, “uv run pre-commit” 명령어를 사용하여 pre commit hook을 실행하세요.
- 오류가 발생하면 작업을 끝내지않고 끝까지 수정합니다.

### key 4. 코드를 쓰지 말고 질문을 쓰자.

코드에 손이 가지 말고, 질문을 잘 작성하는 것이 중요하다.

AI는 거의 반자율주행 모드까지 왔다.

**넓게 시작해서 좁혀나간다**

```jsx
재귀함수를 사용하지 않고, 외부 라이브러리 없이, 캐싱을 사용하여 성능을 최적화한 팩토리얼을 계산하는 계산기를 만들어줘.
-> 계산기를 만들거야. 외부 라이브러리 없이 사칙연산, 재귀함수를 구현해줘. 이후 팩토리얼을 캐싱을 사용하여 최적화해줘.
```

**예제와 함께 오해없게**
입출력 샘플, 경계/실패 케이스 등을 예제로 포함한다.

**작은 단위로 나눈다**
 프롬프트를 단계별로 쪼개서.

### key 5. 리뷰가 곧 품질이다.

---

Copilot command
/explain : 에디터에 열려있거나 선택한 코드가 어떻게 동작하는지 자연어로 풀어주는 기능

/fix : AI에게 해결책을 요청.

/doc : 함수나 클래스에 대한 docstring을 자동 생성.

Copilot Code Review

Review Selection - 내가 선택한 부분을 리뷰

Review Changes - 커밋 전 변경 부분 전체를 리뷰

Ask 모드 : 코드 설명, 아이디어 브레인스토밍, 라이브러리 사용법 등 궁금증 해결

코드 맥락을 읽어 즉석 예시/대안 등 제시.

→ 어떤 라이브러리를 쓰고 싶은데, 어떻게 쓰는지 물어볼 때 사용.

Edit 모드 : 특정 파일을 안전하게 수정하고 싶을때. 변경내용을 diff로 보여주어 Accept 또는 Discard 수행 가능.

Agent 모드

~해줘와 같이 느슨한 목표를 주면 AI가 스스로 설계하고 파일을 만들며 터미널 명령까지 실행하며 목표를 완수하려고 시도한다.

테스트 실패/컴파일오류까지 반복 실행. 필요하면 패키지 설치 빌드 테스트까지 자동으로 수행

토큰은 모든 과정에 사용, 커맨드 명령어 넣는데까지 전부 토큰이다.
